Рассматриваем **один поток (thread)** вашего ядра. Один поток вычисляет размытие **для одного пикселя и для всех его каналов**.

Обозначим:

* ( W = 2 \cdot BLUR_SIZE + 1 )
* При `BLUR_SIZE = 5` → ( W = 11 )
* Размер окна: ( 11 \times 11 = 121 )
* Пусть число каналов = **C** (обычно 3 для RGB)

---

# (1) Сколько операций с плавающей точкой?

Рассматриваем внутренний цикл для **одного канала**.

### Внутри окна 11×11:

Для каждого допустимого соседа:

```cpp
sum += input[idx];
```

Это:

* 1 операция сложения (float)

Таких операций максимум **121**.

После цикла:

```cpp
output[...] = sum / count;
```

* 1 деление (float)

---

### Итого на 1 канал:

* 121 сложение
* 1 деление

= **122 FLOP**

---

### Для C каналов:

[
FLOP = C \cdot (121 + 1)
]

Если RGB (C = 3):

[
3 \cdot 122 = 366 \text{ операций с плавающей точкой на поток}
]

---

# (2) Сколько чтений из глобальной памяти?

Внутри окна:

```cpp
sum += input[idx];
```

Каждый раз происходит:

* 1 чтение из `input[]`

Максимум 121 чтение на канал.

---

### На 1 канал:

121 чтение

### На C каналов:

[
Reads = C \cdot 121
]

Для RGB:

[
3 \cdot 121 = 363 \text{ чтения}
]

---

# (3) Сколько записей в глобальную память?

В конце:

```cpp
output[outIdx] = ...
```

* 1 запись на канал

---

### На C каналов:

[
Writes = C
]

Для RGB:

[
3 записи
]

---

# (4) Какие оптимизации возможны?

Сейчас ядро **очень неоптимально**.

---

## 🔴 1. Главная проблема — глобальная память

Каждый поток:

* Читает 121 × C элементов
* Соседние потоки читают почти те же данные

👉 Данные читаются **многократно** из глобальной памяти.

Это memory-bound kernel.

---

## 🟢 Оптимизация №1: Shared Memory (самая важная)

Загрузить тайл изображения в shared memory:

* Блок 16×16
* С halo зоной ±5 пикселей
* Загружать один раз
* Использовать из shared memory

Тогда:

* Глобальные чтения резко уменьшаются
* Данные переиспользуются внутри блока

Прирост может быть **в 10–30 раз**

---

## 🟢 Оптимизация №2: Разделение свёртки (Separable Blur)

Box blur 11×11 можно разбить на:

1. Горизонтальный blur 11×1
2. Вертикальный blur 1×11

Вместо:

[
11 \times 11 = 121
]

Получаем:

[
11 + 11 = 22
]

👉 В 5.5 раза меньше операций и чтений.

Это **самая мощная математическая оптимизация**.

---

## 🟢 Оптимизация №3: Убрать деление

Так как окно фиксированное (121):

```cpp
float inv = 1.0f / 121.0f;
output = sum * inv;
```

Умножение быстрее деления.

---

## 🟢 Оптимизация №4: Использовать константную память

Если ядро будет использовать маску (для Gaussian blur),
её можно положить в `__constant__` память.

---

## 🟢 Оптимизация №5: Векторизация (float3 / float4)

Обрабатывать RGB сразу:

```cpp
float3 val = ...
```

Это улучшит coalescing.

---

## 🟢 Оптимизация №6: Уменьшить ветвления

Проверки границ:

```cpp
if (nx >= 0 && ...)
```

Можно:

* Либо обрабатывать только внутреннюю область
* Либо расширить изображение padding'ом

Меньше divergence → быстрее.
